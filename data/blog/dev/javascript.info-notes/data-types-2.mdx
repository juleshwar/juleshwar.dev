## Array methods

- Link: https://javascript.info/array-methods
- `splice` can be used to add items at an index in an array (and optionally delete items). We can use negative indices as well to start from the end of the array
    
    ```jsx
    ["Mon", "Tue", "Wed", "Jan", "Sat"].splice(3, 1, "Thu", "Fri") // ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat"]
    
    ["Mon", "Tue", "Wed", "Jan", "Sat"].splice(-2, 1, "Thu", "Fri") // ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat"]
    ```
    
- `at` can be used to access items and it accepts negative indices as well
    
    ```jsx
    [1,2,3,4,5].at(-1) // 5
    ```
    
- We can use `concat` on [array-like](https://www.notion.so/JavaScript-a78c3c235d034f8c91364a37f676fce2?pvs=21) elements, `[Symbol.isConcatSpreadable]` needs to be set to `true` for it to work as expected
    
    
    ```jsx

    	0:"a",
    	1:"b",
    	length: 2
    }
    console.log([].concat(moo)) // [{0: 'a', 1: 'b', length: 2}] âŒ
    ```
    
    ```jsx
    let moo = {
    	0:"a",
    	1:"b",
    	length: 2,
    	[Symbol.isConcatSpreadable]: true
    }
    console.log([].concat(moo)) // ['a', 'b'] âœ…
    ```
    
- `indexOf` doesnâ€™t handle `NaN` correctly because the internal comparison algorithm has a bug. `includes` which was added later to JS handles it correctly
    
    ```jsx

    [NaN].includes(NaN) // true âœ…
    ```
    
- `Array.sort` compares elements as strings by default ðŸ¤¯Â #til
    
    
    ```jsx

    arr.sort();
    console.log( arr );  // [1, 15, 2] âŒ
    ```
    
    ```jsx

    arr.sort((a,b) => +a - +b); // doing +"str" coerces str into a number
    ```

- `reduce` is a great way to *******reduce******* an array to a single value. It is a versatile method which can where you can customise the result. It can mimic `map` , `filter`, `forEach`, `join` etc.
    - `reduceRight` reduces the array from the right
    
    ```jsx
    [1,2,3].join(';') // 1;2;3
    [1,2,3].reduce((acc, curr) => `${acc};${curr}`) // 1;2;3
    [1,2,3].reduceRight((acc, curr) => `${acc};${curr}`) // 3;2;1
    ```

- https://javascript.info/iterable#symbol-iterator
    
    `[Symbol.iterator]()` is the reason why `forâ€¦of` works. When you call forâ€¦of on an object,
    
    1. It calls `[Symbol.iterator]` method once (or errors if not found). The method must return anÂ ***iterator*Â â€“ an object with the methodÂ `next`**.
    2. Onward,Â `for..of`Â worksÂ *only with that returned object*.
    3. WhenÂ `for..of`Â wants the next value, it callsÂ `next()`Â on that object.
    4. The result ofÂ `next()`Â must have the formÂ `{done: Boolean, value: any}`, whereÂ `done=true` means that the loop is finished, otherwiseÂ `value`Â is the next value.
    
    ```jsx

      from: 3,
      to: 8,
    };

        current: this.from,
        last: this.to,
        next() {

            return {
              value: this.current++,
              done: false,
            };
          } else {
            return {
              done: true,
            };
          }
        },
      };
    };
    
    for (let x of obj) {
      console.log(x); // 3,4,5,6,7,8 
    }
    ```
